CHAPTER 1
INTRODUCTION
The use of data-driven decision support systems has increased in many areas of computer engineering, including education, healthcare, finance, cybersecurity, and transportation. In the educational domain, institutions often collect data about attendance, assignments, quiz scores, final grades, and student engagement. However, this data is frequently used only for record keeping, while its potential for early prediction and academic support remains underused. Machine learning provides methods that can learn patterns from historical data and support prediction tasks when the data is prepared and evaluated correctly.

1.1. Literature Review

Machine learning has been widely studied for prediction and classification tasks because it can identify patterns in structured and unstructured data. Traditional statistical models, such as Logistic Regression, are commonly used as baseline methods because their coefficients are easier to interpret and they provide a useful reference point for comparison. More complex models, such as Random Forest and Support Vector Machine, can capture non-linear relationships, but they may require more careful tuning and interpretation.
In educational data mining, previous studies have explored student performance prediction using attendance, assessment marks, demographic information, learning management system activity, and behavioral indicators. These studies generally show that early academic indicators can be useful for identifying students at risk of low performance. However, the quality of prediction depends strongly on the dataset, feature selection, class balance, and evaluation procedure. A model with high accuracy may still be weak if it fails to correctly identify the minority group, such as students at risk.
The main gap addressed by this project is the need for a simple, reproducible, and explainable machine learning workflow that can be understood by undergraduate students and used as a prototype decision-support tool. Instead of focusing only on model accuracy, this project also considers preprocessing steps, comparison of multiple algorithms, evaluation metrics, and ethical handling of student data.

CHAPTER 2
MACHINE LEARNING BACKGROUND
This chapter explains the theoretical concepts required to understand the project. Machine learning is a branch of artificial intelligence in which algorithms learn patterns from data and use those patterns to make predictions or classifications on new data. In this project, supervised learning is used because the dataset contains input features and known target labels.
2.1. Machine Learning Classification
Classification is a supervised learning task where the model assigns an input sample to one of several predefined classes. In a student performance prediction system, the input may include attendance rate, assignment average, quiz score, number of missed classes, and study-hours category. The output class may be Pass/Fail or Low/Medium/High risk. Logistic Regression can be used as a simple baseline classifier, while Random Forest can be used to model more complex relationships between the features and the target label.
2.1.1. Data Preprocessing and Feature Engineering
Data preprocessing is an essential step before training a machine learning model. Missing values may be replaced using mean, median, mode, or a domain-specific value. Categorical features such as gender, department, or study level must be encoded into numerical form. Feature engineering may include creating new variables, such as attendance percentage or assignment completion ratio. Poor preprocessing can lead to biased or unreliable model results even when the chosen algorithm is strong.
2.1.2. Model Evaluation Metrics
Evaluation metrics are used to measure how well the model performs on unseen data. Accuracy measures the overall proportion of correct predictions, but it can be misleading if the dataset is imbalanced. Precision measures how many predicted positive cases are actually positive, while recall measures how many actual positive cases are detected. F1-score balances precision and recall. ROC-AUC summarizes the model ability to separate classes across different thresholds.

The main challenge in this type of project is not only model training, but also data quality. Missing records, inconsistent grade formats, and imbalanced target classes can affect the reliability of the prediction. Another challenge is interpretation. A prediction of "High Risk" should not be treated as a final judgment about a student. Instead, it should be used as an early signal for further academic support.
4.2. Conclusion and Future Work
This project demonstrated how a complete machine learning workflow can be designed for student performance prediction. The project covered dataset preparation, preprocessing, baseline modeling with Logistic Regression, comparison with other classifiers, model evaluation, and prototype dashboard design. The sample system can support academic advising by identifying students who may need attention earlier than traditional manual review.
The main limitations are the dependence on dataset quality, the possibility of biased features, and the limited generalizability of a model trained on a small dataset. Future work may include using a larger dataset, applying cross-validation, adding explainable AI techniques such as SHAP, integrating the system with a learning management system, and testing the prototype with real users under proper privacy and ethical approval procedures.
